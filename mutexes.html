Data races

A data race happens when:

  • Two goroutines access the same memory
  • At least one writes to it
  • The accesses aren't synchronized

 

same goroutine

2fmt.Println(c)

An example: generating unique IDs

1import (
2   "fmt"
3   "sync"
4)

5type IDGenerator struct {
6   prefix string
7   mu     sync.Mutex
8   num    int
9}

10func NewIDGenerator(prefix string) *IDGenerator {
11   return &IDGenerator{prefix: prefix}
12}

13func (g *IDGenerator) NewID() string {
14   g.mu.Lock()
15   defer g.mu.Unlock()
16   g.num++
17   return fmt.Sprintf("%s%d", g.prefix, g.num)
18}
